PORTFOLIO

Kubernetes

Code Contributor
  • Kubernetes is an open source system for automating deployment and management of containers. Originally developed by Google.

    Matt got involved with the community in late 2014 by creating software to install and configure a Kubernetes cluster on Ubuntu Linux.

    Since then I have contributed code and fixed bugs in the Kubernetes project. Attended the 2016 KubeCon in Seattle Washington, and the 2017 KubeCon in Berlin Germany.

Juju

Charmer
  • Juju is an open source modeling tool that makes it easy to deploy software on different public clouds.

    Voted in as a "Charmer" who is responsible to create, maintain and review software projects known as "charms" in the Juju ecosystem.

    A "charm" is DevOps code that install and configure application software in the Juju system. Charms have hooks to react to events in the software lifecycle from install, relation, to scaling, and destruction.

    Instructed several educational sessions known as "charm schools" for the community and business partners to teach developers on how to write software that would perform well in the Juju system.

Jujubox

Maintainer
  • A Docker project used to contain the bare minimum needed for a Juju container. Used in development, and production for isolation, so the CI system does not have to install or upgrade the Juju software. This project created a cleaner system with less dependencies.

 

Simple Password Maker

Developer
  • A personal "for fun" Python project that creates unique passwords based on the hash values of username, URL, and other user inputs. You can change the character set to create passwords for any system.

    This project contains Continuous Integration (CI) tests that verify each pull request passes the tests as they are created. Presented this project at the Python meetup group.

Weather Scheduler

Developer
  • A personal "for fun" Python project written to automate a set of actions taken by a human every day

    Someone from the local bicycle club has to manually look up the weather information, choose a map based on wind direction, and type up an email to the group with all this information the day before the regularly scheduled rides.

    This program uses the requests package to call Weather Underground APIs to get the weather information, Formats a template file and even sends the email to the group.
